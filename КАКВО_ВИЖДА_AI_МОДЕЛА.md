# Какво "Вижда" AI Модела при Анализ на Ириса? 👁️

## Кратък Отговор

**AI модела получава ЧИСТО, ОРИГИНАЛНО изображение на ириса БЕЗ каквато и да е наложена топографска карта или шаблон.**

Това е критично важно за точността на анализа - модела трябва да види истинската структура на ириса без визуални смущения.

---

## Детайлно Обяснение

### 1. Какво Изображение Получава AI Модела? 📷

#### Оригинално Изображение БЕЗ Overlay
```
✅ ЩО ПОЛУЧАВА МОДЕЛА:
├─ Оригинална снимка на ириса (компресирана до ~200KB)
├─ Без топографска карта
├─ Без часовниковидни маркери
├─ Без наложени зони или шаблони
└─ Чиста структура на ириса

❌ КАКВО НЕ ПОЛУЧАВА:
├─ Топографска карта с 12-те зони
├─ Часовниковидни обозначения (12:00, 3:00, и т.н.)
├─ Цветно кодирани зони
└─ Графични overlay елементи
```

#### Технически Детайли на Обработката
```javascript
// От AnalysisScreen.tsx, ред 983-994
// Коментар в кода ЕКСПЛИЦИТНО потвърждава:
"Note: We create composite image with overlay for display purposes only
 The AI analysis will use the ORIGINAL image WITHOUT overlay to avoid visual interference"

// Използва се оригиналното изображение:
const imageForAnalysis = iris.dataUrl  // Това е чистата снимка

console.log(`📷 [ИРИС ${side}] Използване на ОРИГИНАЛНО изображение БЕЗ overlay за AI анализ`)
console.log(`✅ [ИРИС ${side}] Изображението ще се изпрати към AI БЕЗ топографска карта`)
```

### 2. Защо Без Топографска Карта? 🤔

#### Причини за Избор на Чисто Изображение

1. **Предотвратяване на Визуални Смущения**
   - Наложена карта би могла да скрие или изкриви реални структури
   - Линии и маркери биха затруднили откриването на артефакти
   - AI може да бърка графични елементи с реални находки

2. **Позволява на Модела да Използва Собствените си "Очи"**
   - GPT-4o/Gemini имат силни computer vision възможности
   - Модела може сам да идентифицира структури и находки
   - По-добра точност при откриване на фини детайли

3. **Избягва "Подсказки" които Биха Повлияли на Обективността**
   - Наложена зона може да предразположи модела към находки
   - Чистото изображение гарантира обективен анализ

### 3. Как Модела Се Ориентира? 🧭

Макар че модела НЕ получава визуална топографска карта, той **е инструктиран** чрез текстов prompt как да интерпретира позициите в ириса.

#### A. Топографска Карта в Текстовия Prompt

```
От default-prompts.ts:

ИРИСОВА_ТОПОГРАФИЯ(12h=връх,часовн_посока):
12h(0-30°) → мозък/ЦНС
1-2h(30-90°) → щ.жлеза/ендокр
3h(90-120°) → бял_дроб(R)
4h(120-150°) → черен_дроб
5-6h(150-210°) → стомах/панкреас
7-8h(210-270°) → черва
9h(270-300°) → урогенит(L)
10h(300-330°) → бъбреци
11h(330-360°) → далак/имун
```

#### B. Координатна Система

Модела е инструктиран да използва **часовников подход**:
- **12:00 (0°)** = Връх на ириса (най-горе)
- **3:00 (90°)** = Дясна страна
- **6:00 (180°)** = Долу
- **9:00 (270°)** = Лява страна
- **Посока**: Часовникова стрелка (clockwise)

#### C. Пример: Как Модела "Вижда" и Локализира

```
ВИЗУАЛНО на изображението:
👁️ Модела вижда: "Тъмна овална структура (лакуна) 
                  в горен-десен квадрант на ириса"

ИНТЕРПРЕТАЦИЯ чрез prompt:
📍 Позиция: Горен-десен = около 1:00-2:00 (30-90°)
📖 Справка в топографията: 1-2h → Щитовидна жлеза/Ендокринна
✅ Резултат: "Лакуна в ендокринна зона (1:00-2:00)"
```

### 4. Как Модела Открива Артефакти? 🔍

#### A. Диагностична Проверка (Първа Стъпка)

Преди основния анализ, системата прави **диагностична проверка** - пита модела свободно какво вижда:

```javascript
// От AnalysisScreen.tsx, ред 1005-1027
const diagnosticPrompt = `Ти си експертен иридолог. 
Получаваш чисто изображение на ${sideName} ирис БЕЗ топографска карта.

ЗАДАЧА: Опиши в СВОБОДЕН ТЕКСТ (4-6 изречения) какво РЕАЛНО виждаш:

1. КАЧЕСТВО НА ИЗОБРАЖЕНИЕТО:
   - Можеш ли ясно да видиш ириса и неговата структура?
   - Има ли проблеми с осветлението, фокуса или резолюцията?

2. ВИДИМИ СТРУКТУРНИ НАХОДКИ в ириса:
   - Виждаш ли лакуни (тъмни процепи/празнини в тъканта)?
   - Виждаш ли крипти (малки дълбоки дупки)?
   - Виждаш ли пигментни петна (кафяви, оранжеви, жълти)?
   - Виждаш ли радиални линии (от центъра навън)?
   - Виждаш ли концентрични пръстени или промени в плътността?

3. ЗАКЛЮЧЕНИЕ:
   - Изглежда ли изображението подходящо за пълноценен иридологичен анализ?
   - Виждаш ли НЯКАКВИ проблемни находки или зони?`
```

Този отговор се **записва и показва** в логовете за диагностика.

#### B. Типове Артефакти които Модела Търси

От `default-prompts.ts` и `airis-knowledge.ts`:

```
ТИПОВЕ_НАХОДКИ_И_ЗНАЧЕНИЕ:
├─ Лакуни → тъмни_процепи_груби_ръбове → слабост_орган/тъкан
├─ Крипти → малки_тъмни_дупки → хрон_стрес/възпал
├─ Пигменти → кафяви/жълти_петна → токс_натоварване/черен_дроб
├─ Радиални_линии → център→периф → нервен_стрес/надбъбречна_умора
├─ Автоном_пръстен → кръг_около_зеница → храносм/ANS_дисбаланс
└─ Arcus_senilis → бял/сив_периф → липиден_дисметаб/СС_риск
```

#### C. Критично Правило: Игнориране на Отражения

```
КРИТИЧНО: отличи_светл_отражения_vs_реални_артефакти
(отражения = ярко_бяло, остри_ръбове, симетрични → НЕ_артефакти)

ИГНОРИРАЙ:
├─ Ярки бели отражения
├─ Огледални ефекти
├─ Светлинни точки
└─ Симетрични блясъци
```

### 5. Структура на Анализа (Поетапно) 📋

#### Pipeline v9 - Многостъпков Процес

Системата използва **Pipeline v9** - модерен подход с няколко стъпки:

##### **Стъпка 1: Геометрична Калибрация** 🎯
```
ЦЕЛ: Оцени качеството и установи координатна система

Модела проверява:
├─ Фокус (добър/среден/лош)
├─ Светлинни отражения (няма/малко/средно/много)
├─ Запушване (няма/малко/средно/много)
├─ Детектира границите на ириса (limbus)
├─ Установява център и посоки (12:00, 3:00, и т.н.)
└─ Определя 12 концентрични пръстена (от зеница до периферия)

Изход:
{
  "ok": true,
  "quality": { "score0_100": 85, "focus": "good", "glare": "low" },
  "geo": { "mins": 60, "rings": 12, "degPerMin": 6 }
}
```

##### **Стъпка 2A: Откриване на Структурни Находки** 🔬
```
ЦЕЛ: Открий структурни аномалии БЕЗ диагноза

Модела търси:
├─ Лакуни (lacuna)
├─ Крипти (crypt)
├─ Гигантски лакуни (giant_lacuna)
├─ Атрофични области (atrophic_area)
├─ Радиални бразди (radial_furrow)
├─ Дълбоки радиални цепки (deep_radial_cleft)
├─ Трансверзални влакна (transversal_fiber)
└─ Структурна асиметрия (structural_asymmetry)

За всяка находка записва:
{
  "type": "lacuna",
  "minuteRange": [15, 20],  // Позиция по часовник (минути 0-59)
  "ringRange": [4, 6],       // Пръстен (от зеница към периферия)
  "size": "m",               // xs, s, m, l
  "notes": "Овална структура с неравни ръбове",
  "confidence": 0.85
}
```

##### **Стъпка 2B: Откриване на Пигменти и Пръстени** 🎨
```
ЦЕЛ: Открий цветни промени и периферни знаци

Модела търси:
├─ Пигментни петна (orange_rust|brown_black|yellow)
├─ Пигментни облаци (diffuse haze)
├─ Пигментни ленти (belt/arc)
├─ Brushfield-подобни петна
├─ Нервни пръстени (nerve_rings)
├─ Лимфатичен розариум (lymphatic_rosary)
├─ Scurf rim (тъмен периферен пръстен)
└─ Sodium ring (бледо-млечен пръстен)
```

##### **Стъпка 3: Биологична Интерпретация** 🧬
```
ЦЕЛ: Интерпретирай находките в контекст на здравето

Модела корелира:
├─ Структурни находки (от стъпка 2A)
├─ Пигментни находки (от стъпка 2B)
├─ Въпросник (възраст, пол, оплаквания, навици)
├─ Топографска карта (коя зона = кой орган)
└─ База знания (какво означават находките)

Генерира:
├─ 12 зони със статус (normal/attention/concern)
├─ Общо здравословно състояние (0-100)
├─ 6 системни оценки (храносмилателна, имунна, нервна, и т.н.)
└─ Препоръки (храна, добавки, начин на живот)
```

### 6. Ключови Принципи на Инструктиране 📜

#### A. Обективност
```
⚠️ КРИТИЧНО ПРАВИЛО - ОБЕКТИВЕН АНАЛИЗ:
АНАЛИЗИРАЙ САМО ТОВА, КОЕТО РЕАЛНО ВИЖДАШ НА СНИМКАТА!
НЕ създавай "виртуални" находки базирани на въпросника.
```

#### B. Логика за Статуси на Зоните
```
ЛОГИКА ЗА МАРКИРАНЕ НА ЗОНИ:
1. ЯСНА НАХОДКА + СЪОТВЕТЕН СИМПТОМ → status:"concern" | priority:high
2. ЯСНА НАХОДКА БЕЗ СИМПТОМ → status:"attention" | priority:medium
3. БЕЗ НАХОДКА + СИМПТОМ → status:"normal" (НЕ маркирай като проблем)
4. БЕЗ НАХОДКА БЕЗ СИМПТОМ → status:"normal"
```

#### C. Описване на Находки
```
findings ТРЯБВА ДА ОПИСВА САМО ВИЗУАЛНО ВИДИМОТО:
✅ "Три ясни лакуни в горния сектор"
✅ "Тъмно пигментно петно около 4h позиция"
✅ "Визуално чиста зона без забележими находки"

❌ "Клиентът има гастрит" (това е симптом, не визуална находка)
❌ "Вероятно проблеми с..." (интерпретацията е отделно)
```

### 7. Примерен Workflow 🔄

```
СТЪПКА 1: КАЧВАНЕ НА ИЗОБРАЖЕНИЕ
├─ Потребител качва снимка на ирис
├─ Компресия до ~200KB (запазване на качество)
└─ БЕЗ добавяне на overlay или маркери

СТЪПКА 2: ПОДГОТОВКА ЗА AI
├─ Зарежда се топографска карта (ТЕКСТОВА, не визуална)
├─ Зарежда се база знания за артефакти
├─ Подготвя се prompt с инструкции
└─ Изображението остава ЧИСТО

СТЪПКА 3: ДИАГНОСТИЧНА ПРОВЕРКА
├─ AI получава изображението
├─ Описва свободно какво вижда
├─ Потвърждава качество и видими находки
└─ Резултат се записва в лога

СТЪПКА 4: ГЕОМЕТРИЧНА КАЛИБРАЦИЯ (Pipeline v9)
├─ AI установява координатна система
├─ Детектира граници на ириса
├─ Оценява качество (фокус, отражения)
└─ Връща геометрична конфигурация

СТЪПКА 5: ОТКРИВАНЕ НА СТРУКТУРИ
├─ AI сканира за лакуни, крипти
├─ Записва позиция в минути и пръстени
├─ Оценява размер и увереност
└─ Игнорира светлинни отражения

СТЪПКА 6: ОТКРИВАНЕ НА ПИГМЕНТИ
├─ AI търси цветни промени
├─ Идентифицира пигментни петна
├─ Открива периферни знаци
└─ Записва тип и локация

СТЪПКА 7: БИОЛОГИЧНА ИНТЕРПРЕТАЦИЯ
├─ AI свързва находки с топографска карта
├─ Корелира с въпросника
├─ Определя статус на зоните
├─ Генерира системни оценки
└─ Създава препоръки

СТЪПКА 8: ГЕНЕРИРАНЕ НА ДОКЛАД
├─ Обобщава всички находки
├─ Приоритизира по важност
├─ Създава визуализации с overlay (САМО за показване)
└─ Представя на потребителя
```

### 8. Визуализации (Само за Показване) 🖼️

**ВАЖНО**: Топографската карта и цветно кодираните зони се създават **СЛЕД** анализа, само за **визуално представяне** на резултатите.

```
КАКВО ВИЖДА ПОТРЕБИТЕЛЯ (в доклада):
├─ Ирис с наложена топографска карта (12 зони)
├─ Цветно кодиране:
│   ├─ Зелено → normal (норма)
│   ├─ Жълто → attention (внимание)
│   └─ Червено → concern (притеснение)
├─ Часовникови маркери (12:00, 3:00, и т.н.)
└─ Текстови описания на находки

КАКВО Е ВИДЯЛ AI МОДЕЛА:
└─ Само чистото изображение БЕЗ overlay
```

### 10. Често Задавани Въпроси ❓

#### Q1: Защо модела не вижда някои находки?
```
A: Възможни причини:
1. Компресията е намалила детайла (агресивна до ~200KB)
2. Ниско качество на оригиналната снимка
3. Силни светлинни отражения
4. Модела е консервативен при несигурни находки
5. Находката е описана в диагностичния текст, но не е парсирана в JSON
```

#### Q2: Има ли начин да видим "очите на AI"?
```
A: ДА! Диагностичната проверка:
1. Включена по подразбиране (enableDiagnostics: true)
2. AI описва свободно какво вижда
3. Резултата се записва в diagnosticResponses
4. Показва се в Debug секцията на екрана
5. Може да се сравни с финалния JSON резултат
```

#### Q3: Топографската карта надеждна ли е?
```
A: ДА, базирана на класическа иридология:
├─ 12-часовникова схема (Jensen, Sharan)
├─ Ляво/дясно око различия
├─ Проверена от практикуващи иридолози
└─ Модела е обучен с обширни данни
```

#### Q4: Може ли да подобрим точността?
```
A: ДА, няколко начина:
1. Качване на по-високо качество снимки (600x600+)
2. Добро осветление, минимум отражения
3. Увеличаване на размера на изображението до 400-600KB
4. Използване на gpt-4o (най-добър за vision)
5. Добавяне на допълнителен контекст в Admin → AI Prompt
```
